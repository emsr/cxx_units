Mass Class

Terminology

None.

ConOps

Represent the abstract concept of a mass. The internal representation of the units is hidden from the user. Functions include comparison, arithmetic, and interaction operators. Methods provided when necessary to interact with related classes. Functions provided to read and write an mass object in binary format. A function providing a self-test is provided. A set of common subclasses are provided.

Interfaces with this class are shown at the Unit Library Service level.

Implementation Details

Mass is one class in a set of classes providing basic unit objects needed for a variety of applications.

When a value is assigned to a Mass object, the value is converted into the internal representation units. When a value is requested from an object, it is converted into the requested units.

Subclasses defined for convenience are KilogramsMass and PoundsMass.

Object Scenarios / Test Procedures

An object of Mass can be declared without a value. When a value is assigned to that object, it must be defined in terms of a subclass to Mass. When an object of a Mass class is declared and is given a value, that value must be specified in terms of some pre-defined subclass. This is the only way that the Mass class can know how the number provided should be interpreted.

A function is provided to convert a Mass object to a double. This also requires a subclass specification so the caller can be assured of getting the result in an expected way that is not dependent on the internal representation.

To test this class, simply run the unitsTest.exe program by typing "unitsTest Mass". The test routine will indicate if the class passed it's self-test.

Requirements

Provide arithmetic operators, as appropriate, to add, subtract, multiply, and divide objects of this class.

Provide operators to interact appropriately with doubles.

Provide a way to print the value of an object.

Provide binary read and write functions.
