Area Class

Terminology

None.

ConOps

Represent the abstract concept of a area. The internal representation of the units is hidden from the user. Functions include comparison, arithmetic, and interaction operators. Methods provided when necessary to interact with related classes. Functions provided to read and write an area object in binary format. A function providing a self-test is provided. A set of common subclasses are provided.

Interfaces with this class are shown at the Unit Library Service level.

Implementation Details

Area is one class in a set of classes providing basic unit objects needed for a variety of applications.

When a value is assigned to a Area object, the value is converted into the internal representation units. When a value is requested from an object, it is converted into the requested units.

Subclasses defined for convenience are MetersArea, KilometersArea, NauticalMilesArea, StatuteMilesArea, and AcresArea.

Object Scenarios / Test Procedures

An object of Area can be declared without a value. When a value is assigned to that object, it must be defined in terms of a subclass to Length. When an object of a Area class is declared and is given a value, that value must be specified in terms of some pre-defined Length subclass. This is the only way that the Area class can know how the numbers provided should be interpreted.

A function is provided to convert a Area object to a double. This also requires a Length subclass specification so the caller can be assured of getting the result in an expected way that is not dependent on the internal representation.

To test this class, simply run the unitsTest.exe program by typing "unitsTest Area". The test routine will indicate if the class passed it's self-test.

Example Code Snippet

      Length nsDistance = NauticalMilesLength(5.0);
      Length ewDistance = NauticalMilesLength(2.0);

      Area threatArea = nsDistance*ewDistance;

      if (((double)AreaFormat(threatArea) >
           MIN_AREA) &&
          ((double)AreaFormat(threatArea) <=
          MEDIUM_AREA)
         )
      {
        threatSearchFidelity = medium;
      }

Requirements

Provide comparison operators, as appropriate, to compare to objects of this class.

Provide arithmetic operators, as appropriate, to add, subtract, multiply, and divide objects of this class.

Provide operators to interact appropriately with the Area class and doubles.

Provide a way to print the value of an object.

Provide binary read and write functions.

System Dependencies

Required classes include:

  • Unit
    • This class inherits from the Unit class.
  • Length
    • Lengths are combined to produce an Area.
