Acceleration Class

Terminology

None.

ConOps

Represent the abstract concept of a acceleration. Functions include comparison, arithmetic, and interaction operators. Methods provided when necessary to interact with related classes. Functions provided to read and write an acceleration object in binary format. A function providing a self-test is provided. A set of common subclasses are provided.

Interfaces with this class are shown at the Unit Library Service level.

Implementation Details

Acceleration is one class in a set of classes providing basic unit objects needed for a variety of applications.

Subclasses defined are MetersSecondAcceleration, FeetSecondAcceleration, GsAcceleration.

Object Scenarios / Test Procedures

An object of Acceleration can be declared without a value. When a value is assigned to that object, it must be defined in terms of a Length subclass and a Time subclass. When an object of a Acceleration class is declared and is given a value, that value must also be specified in terms of a Length subclass and a Time subclass. This is the only way that the Acceleration class can know how the value provided should be interpreted.

A function is provided to convert a Acceleration object to a double. This requires a Length subclass and a Time subclass specification so the caller can be assured of getting the result in an expected way that is not dependent on the internal representation.

To test this class, simply run the unitsTest.exe program by typing "unitsTest Acceleration". The test routine will indicate if the class passed it's self-test.

      Acceleration acceleration;

      acceleration = route.legAcceleration (pt1, pt2);

      }

Requirements

Provide comparison operators, as appropriate, to compare to objects of this class.

Provide arithmetic operators, as appropriate, add, subtract, multiply, and divide objects of this class.

Provide operators to interact appropriately with the Time class, the Speed class, and doubles.

Provide a way to print the value of an object.

Provide binary read and write functions.

System Dependencies

Required classes include:

  • Unit
    • This class inherits from the Unit class.
  • Time
    • Time is used in Acceleration operators.
  • Speed
    • Speed is used in Acceleration operators.
