Density Class

Terminology

None.

ConOps

Represent the abstract concept of density. The internal representation of the units is hidden from the user. Functions include comparison, arithmetic, and interaction operators. Methods provided when necessary to interact with related classes. Functions provided to read and write a density object in binary format. A function providing a self-test is provided.

Interfaces with this class are shown at the Unit Library Service level.

Implementation Details

Density is one class in a set of classes providing basic unit objects needed for a variety of applications.

Object Scenarios / Test Procedures

An object of Density can be declared without a value. When a value is assigned to that object, it must be defined in terms of a Mass subclass and a Volume subclass. When an object of a Density class is declared and is given a value, that value must also be specified in terms of a Mass subclass and a Volume subclass. This is the only way that the Density class can know how the number provided should be interpreted.

A function is provided to convert a Density object to a double. This also requires subclass specifications so the caller can be assured of getting the result in an expected way that is not dependent on the internal representation.

To test this class, simply run the unitsTest.exe program by typing "unitsTest Density". The test routine will indicate if the class passed it's self-test.

Example Code Snippet

      Density density;
      Volume  tankCapacity = USGallonsVolume(200);
      Mass    fuelWeight;

      density = gasoline.density();

      fuelWeight = tankCapacity * density;

      cout << "weight = " << fuelWeight << endl;

Requirements

Provide comparison operators, as appropriate, to compare to objects of this class.

Provide arithmetic operators, as appropriate, to add, subtract, multiply, and divide objects of this class.

Provide operators to interact appropriately with the Mass class, the Volume class, and doubles.

Provide a way to print the value of an object.

Provide binary read and write functions.

System Dependencies

Required classes include:

  • Unit
    • This class inherits from the Unit class.
  • Mass
    • Mass is used in Density operators.
  • Volume
    • Volume is used in Density operators.
